# Carding

Turn documents into high-quality, atomic flashcards with AI, and study them on an
Anki-style spaced-repetition schedule (FSRS). See `docs/` for the spec — start with
`docs/VISION.md` and `docs/CARD-QUALITY.md`.

## Setup

```bash
pnpm install
cp .env.example .env.local   # then fill in the values below
```

Required environment (`.env.local`, gitignored):

| Var | What |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY` | Supabase project (manual prereq — see `docs/BUILD-PLAN.md`) |
| `ANTHROPIC_API_KEY` | Card generation |
| `NEXT_PUBLIC_VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`, `VAPID_SUBJECT` | Web Push for reminders — generate with `pnpm gen:vapid` |
| `CRON_SECRET` | Shared secret guarding the reminder cron |

```bash
pnpm dev          # http://localhost:3000
pnpm typecheck    # tsc --noEmit
pnpm build        # production build
```

## PWA (installable app)

Carding is an installable PWA — one codebase for desktop and mobile, no Electron
(offline review is v2).

- **Manifest:** `app/manifest.ts` → served at `/manifest.webmanifest` (Next injects the
  `<link>`). Icons in `public/icons/` are generated by `pnpm gen:icons` (renders SVG via
  headless Chromium — no extra image dependency).
- **Service worker:** `public/sw.js` (registered in `app/layout.tsx`) handles push +
  notification clicks. `next.config.ts` serves it with `no-store` + a strict CSP.
- **Install:** desktop Chrome/Edge show an install button automatically. On iPhone/iPad
  (iOS 16.4+), Share → **Add to Home Screen**; push works from the installed app.

## Daily study reminders

A push notification when you have cards due, at a time you choose (`/settings`).

1. On `/settings`, **Enable notifications** (subscribes this device) and set a reminder
   time. Preferences + subscriptions are stored in Supabase Auth `user_metadata`
   (see the note in `lib/push/types.ts`).
2. Schedule the cron to hit `/api/cron/reminders` every 5–15 min (Supabase pg_cron +
   pg_net, Vercel Cron, GitHub Actions, …). It's idempotent per local day — it fires once
   when the clock passes your time *and* you have due cards.

   ```bash
   curl -H "Authorization: Bearer $CRON_SECRET" https://<host>/api/cron/reminders
   ```

   Returns `{ ok, scanned, notified, pushesSent, pruned, skipped }`.

## Verifying locally

```bash
# PWA assets, SW headers, SW registration, cron auth gating (no real backend needed):
pnpm dev --port 3999 &
BASE_URL=http://localhost:3999 node scripts/walk-pwa.mjs

# End-to-end product loop (needs real Supabase + Anthropic keys):
node scripts/walk.mjs
```

To verify push **delivery** on a real device: serve over HTTPS (`next dev
--experimental-https` or a deployment), install the PWA, enable notifications on
`/settings`, then use **Send a test** — or set your reminder time to now and hit the cron.
